<task type="auto">
  <name>Task 2: Fix stored XSS in all JS template literals (finding 4)</name>
  <files>templates/index.html, templates/db_settings.html</files>
  <action>
1. **Add escapeHtml helper** at the top of the script block in index.html (before any function that uses it), and also in db_settings.html:
   ```javascript
   function escapeHtml(str) {
     if (str == null) return '';
     return String(str).replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;').replace(/'/g,'&#039;');
   }
   ```

2. **index.html** — In each function, wrap ALL user-supplied data (player names, reasons, score details) with `escapeHtml()`:
   - `updatePyramid`: `${escapeHtml(player.name)}` in the player-info div
   - `updatePlayerDropdowns`: `${escapeHtml(p.name)}` in option text for both selects
   - `updateActiveChallenges`: `${escapeHtml(c.challenger_name)}` and `${escapeHtml(c.opponent_name)}`
   - `updateBlockedPlayers`: `${escapeHtml(p.name)}` for both challenger and opponent lists
   - `updateUnavailablePlayers`: `${escapeHtml(p.name)}` and `${escapeHtml(p.unavailability_reason)}`
   - `updateCompletedChallenges`: `${escapeHtml(c.challenger_name)}`, `${escapeHtml(c.opponent_name)}`, `${escapeHtml(c.score_details)}`

3. **db_settings.html**:
   - `showAlert`: Change `$alert.html(message)` to `$alert.text(message)` (alert messages are plain text, no HTML needed)
   - `loadPlayers`: Wrap `${player.name}` with `${escapeHtml(player.name)}` in all 3 occurrences (name display span, input value, password button data attribute). Also wrap `${player.username}` with `${escapeHtml(player.username)}`.

Do NOT change numeric values (id, rank) or system-generated strings (dates, formatted deadlines) — only user-editable text fields.
  </action>
