## Directory Purposes

**Root Directory:**
- Purpose: Project entry point and configuration
- Contains: Main application file, database file, configuration files, git metadata
- Key files: `app.py` (entire backend), `schema.sql` (database structure), `.env` (secrets)

**templates/:**
- Purpose: Server-side rendered HTML templates
- Contains: Jinja2 template files consumed by Flask `render_template()`
- Key files: `index.html` (main UI), `admin.html` (administration)
- Auto-reloaded: Yes (when `FLASK_DEBUG=true`)

**static/:**
- Purpose: Static assets served directly by Flask without processing
- Contains: PNG images used in templates and CSS backgrounds
- Key files: PNG logo images (~1MB each)
- Caching: Browser-cached via HTTP headers

**documentation/:**
- Purpose: Supporting materials and deployment notes
- Contains: Setup guides, deployment instructions
- Not directly used by application code

**.planning/codebase/:**

## Naming Conventions

**Files:**

- Python: `app.py` (lowercase underscore, single main file pattern)
- HTML templates: `snake_case.html` (e.g., `login_tennis.html`, `db_settings.html`)
- Images: Prefixed with sequence number (e.g., `00_moebiusschleife_round.png`, `01_tennis_racket.png`)
- SQL: `schema.sql` (lowercase, .sql extension)
- JSON: `initial_players.json` (camelCase filename)

**Python Functions/Classes:**

- **Functions**: `snake_case` — `get_db()`, `eligible_opponents_for()`, `get_active_challenges()`
- **Classes**: `PascalCase` — `User`, `DataCache`, `CustomJSONEncoder`
- **Constants**: `UPPER_SNAKE_CASE` — `APP_VERSION`, `MAX_LOGIN_ATTEMPTS`, `LOGIN_LOCKOUT_TIME`
- **Routes**: lowercase with underscores — `/get_players`, `/eligible_opponents`, `/submit_result`

**Database:**

- **Tables**: plural, lowercase — `players`, `challenges`
- **Columns**: snake_case — `challenger_id`, `block_opponent_until`, `privilege_level`
- **Views**: descriptive, lowercase — `completed_challenges_view`
- **Indexes**: prefixed with `idx_` — `idx_players_rank`, `idx_challenges_resolved`

**HTML/Template Variables:**

- **Template context**: lowercase with underscores — `current_user`, `players_pyramid`, `active_challenges`
- **JavaScript variables**: camelCase in inline `<script>` blocks (Bootstrap/jQuery conventions)

## Where to Add New Code

**New Feature (Business Logic):**
- Primary code: Add function directly in `app.py` near related functions
  - Challenge-related logic → near `eligible_opponents_for()` (line 667)
  - Player management → near `rerank_players()` (line 765)
  - Data retrieval → near `get_active_challenges()` (line 459)
- API endpoint: Add `@app.route()` function in routes section (lines 1024+)
- Real-time broadcast: Call `emit_data_update()` (line 957) after state mutation

**New Component/Module:**
- Current architecture: Keep in `app.py` (monolithic pattern)
- If refactoring: Create module (e.g., `challenge_logic.py`), import in `app.py`
- Consider: 63 functions/classes already in single file — modularization at 2500+ lines

**New Template/View:**
- HTML template: Create `.html` file in `templates/` directory
- Route handler: Add `@app.route()` function with `render_template()` call
- Styling: Use existing Bootstrap 5 classes in HTML (no separate CSS file in static/)
- JavaScript: Inline `<script>` blocks in template or reference Socket.IO client library

**New Utilities/Helpers:**
- Serialization helpers: Place near `serialize_player()` (line 259) and `serialize_challenge()` (line 286)
- Database helpers: Place in "Data Access" section near `get_db()` (line 368)
- Time utilities: Place near `get_current_time()` (line 355)
- Eligibility/ranking: Place near `eligible_opponents_for()` (line 667)

**Admin Routes:**
- Location: Lines 2075-2492
- Pattern: Use `@admin_required` or `@superadmin_required` decorators
- Examples: `/add_player`, `/update_player`, `/delete_player`, `/reset_database`

**API Endpoints (JSON responses):**
- Location: Lines 1116-1365
- Pattern: Return `jsonify()` instead of template render
- Examples: `/api/realtime_data`, `/get_players`, `/eligible_opponents`
