Create `build-and-push.sh` (chmod +x) with the following structure and exact behavior:

**1. Shebang and strict mode:**
```bash
#!/usr/bin/env bash
set -euo pipefail
```

**2. Configuration constants:**
```bash
REGISTRY="ghcr.io"
NAMESPACE="ainxtgendev"
APP_IMAGE="${REGISTRY}/${NAMESPACE}/stecher-tennis-app"
CADDY_IMAGE="${REGISTRY}/${NAMESPACE}/stecher-tennis-caddy"
BUILDER_NAME="multiarch"
PLATFORMS="linux/amd64,linux/arm64"
```

**3. GHCR_TOKEN validation:**
Check that `GHCR_TOKEN` env var is set and non-empty. If missing, print error message explaining what scope is needed and exit 1:
```
ERROR: GHCR_TOKEN environment variable is not set.
Create a GitHub Personal Access Token with 'write:packages' scope:
  https://github.com/settings/tokens/new?scopes=write:packages
Then export it: export GHCR_TOKEN=ghp_...
```

**4. Version extraction:**
Extract version from `templates/index.html` using:
```bash
VERSION=$(grep -oP 'Version:\s*\K[0-9]+\.[0-9]+' templates/index.html)
```
If `VERSION` is empty, print error and exit 1.

**5. QEMU binfmt registration (idempotent):**
```bash
echo "==> Registering QEMU binfmt handlers..."
docker run --privileged --rm tonistiigi/binfmt --install all
```

**6. Buildx builder creation (idempotent):**
```bash
echo "==> Setting up buildx builder '${BUILDER_NAME}'..."
docker buildx create --name "${BUILDER_NAME}" --driver docker-container --bootstrap --use 2>/dev/null || \
    docker buildx use "${BUILDER_NAME}"
```

**7. GHCR login:**
```bash
echo "==> Logging in to ${REGISTRY}..."
echo "${GHCR_TOKEN}" | docker login "${REGISTRY}" -u "${GHCR_USER:-ainxtgendev}" --password-stdin
```

**8. Build and push app image:**
```bash
echo "==> Building and pushing ${APP_IMAGE}:v${VERSION} (${PLATFORMS})..."
docker buildx build \
    --platform "${PLATFORMS}" \
    --tag "${APP_IMAGE}:latest" \
    --tag "${APP_IMAGE}:v${VERSION}" \
    --push \
    .
```

**9. Build and push caddy image:**
```bash
echo "==> Building and pushing ${CADDY_IMAGE}:v${VERSION} (${PLATFORMS})..."
docker buildx build \
    --platform "${PLATFORMS}" \
    --tag "${CADDY_IMAGE}:latest" \
    --tag "${CADDY_IMAGE}:v${VERSION}" \
    --push \
    -f Dockerfile.caddy \
    .
```

**10. Summary output:**
```bash
echo ""
echo "===== Build Complete ====="
echo "App:   ${APP_IMAGE}:v${VERSION}"
echo "Caddy: ${CADDY_IMAGE}:v${VERSION}"
echo "Tags:  :latest, :v${VERSION}"
echo "Platforms: ${PLATFORMS}"
echo ""
echo "To deploy on RPi: docker compose pull && docker compose up -d"
```

Make the file executable: `chmod +x build-and-push.sh`

The resulting file must look like this (preserving all other lines exactly as-is):

```yaml
services:
  app:
    image: ghcr.io/ainxtgendev/stecher-tennis-app:latest
    networks:
      - tennis_net
    volumes:
      - tennis_data:/app/data
    env_file: .env
    restart: unless-stopped
    # No ports: block — port 5000 only reachable inside Docker network (COMP-02)

  caddy:
    image: ghcr.io/ainxtgendev/stecher-tennis-caddy:latest
    networks:
      - tennis_net
    ports:
      - "${HTTPS_PORT:-10443}:${HTTPS_PORT:-10443}"
      - "80:80"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro
      - caddy_data:/data
    env_file: .env
    dns:
      - 8.8.8.8
      - 8.8.4.4
    restart: unless-stopped
    depends_on:
      app:
        condition: service_healthy

networks:
  tennis_net:
    driver: bridge

volumes:
  tennis_data:   # SQLite persistence — survives compose down (COMP-03)
  caddy_data:    # Caddy state — needed for Phase 3 TLS certs
```

**Create docker-compose.build.yml:**

Create a new override file for local builds on NUC8. Usage comment at top:

```yaml
# Local build override — use with:
#   docker compose -f docker-compose.yml -f docker-compose.build.yml up -d --build

  app:
    build: .
  caddy:
    build:
      context: .
      dockerfile: Dockerfile.caddy
```

This file only overrides the `build:` directives; all other config (networks, volumes, ports, env_file, restart, depends_on) comes from the base docker-compose.yml.
