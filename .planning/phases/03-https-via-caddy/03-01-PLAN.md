Replace the entire Caddyfile content with the following HTTPS configuration. This replaces the Phase 2 `:80` plain HTTP setup.

**ACME_CA handling decision (Claude's discretion per CONTEXT.md):** Using Option B from research -- `ACME_CA` is ALWAYS required to be set. For staging: `https://acme-staging-v02.api.letsencrypt.org/directory`. For production: `https://acme-v02.api.letsencrypt.org/directory`. This avoids the parse error when `{$ACME_CA}` is empty. The `.env.example` (Task 3) documents both URLs.

Write this exact content to `Caddyfile`:

```
# Phase 3: HTTPS reverse proxy with DuckDNS DNS-01 ACME
# Env vars: DUCKDNS_DOMAIN, DUCKDNS_TOKEN, ACME_EMAIL, ACME_CA, HTTPS_PORT
{
	acme_ca {$ACME_CA}
	email   {$ACME_EMAIL}
}

{$DUCKDNS_DOMAIN}.duckdns.org:{$HTTPS_PORT} {
	tls {
		dns duckdns {$DUCKDNS_TOKEN}
	}

	log {
		output stderr
		level ERROR
	}

	reverse_proxy app:5000 {
		header_up Host              {http.request.host}
		header_up X-Real-IP         {http.request.remote}
		header_up X-Forwarded-For   {http.request.remote}
		header_up X-Forwarded-Proto {http.request.scheme}
	}
}

:80 {
	redir https://{$DUCKDNS_DOMAIN}.duckdns.org:{$HTTPS_PORT}{uri} permanent
}
```

Key design points:
- Global options block with `acme_ca` and `email` -- both always set via env vars
- Main HTTPS site block uses `{$DUCKDNS_DOMAIN}.duckdns.org:{$HTTPS_PORT}` (env var domain + port)
- TLS block uses `dns duckdns {$DUCKDNS_TOKEN}` for DNS-01 challenge
- Reverse proxy block preserved from Phase 2 (all four header_up directives for Flask ProxyFix)
- WebSocket: no special config needed -- Caddy v2 reverse_proxy handles upgrade automatically
- Separate `:80` block for HTTP-to-HTTPS redirect (MUST be manual because auto-redirect omits non-standard port)
- Redirect is `permanent` (301) and includes the `:{$HTTPS_PORT}` in the URL
- Use TABS for Caddyfile indentation (Caddy convention), not spaces
